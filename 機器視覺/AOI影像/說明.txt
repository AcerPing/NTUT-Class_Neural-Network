=> 如果不確定是 檔案是 NG 還是 OK 
=> 請直接複製檔名去用 F5 查詢 該圖片屬於 NG 或是 OK
=> 如果你寫的演算法 造成誤判也沒有關係 只要總誤判率低於10% 即可 => 也就是 你有10%的機會 將NG判為OK 或是 OK判為NG


	CAM => 黃色的部分是設計稿 也就是原本銅線或銅面要出現的地方
	    => 黑色的部分是 絕對不能有銅線出現的地方

	OK 或 NG 的資料夾裡面 橘紅色的圖片檔案 就是 真實電路板拍照的圖片 要用來跟設計稿CAM圖片做比對



 NG 裡面包含 設計稿 CAM 跟 原始電路板照片 + 有問題(Problem)的照片 = 廠商判定是 NG 但是 我們認為可以判為 OK 的照片
 	
	NG Perfect  = 所有 NG 的電路板照片

	NG Perfect CAM  = 所有 NG 的電路板設計稿 

	=> 檢測模型的方式 就是更改路徑 使用 NG Perfect & NG Perfect CAM 來當作路徑 => 再告訴我 比對完的結果
	 也就是 去試試看 你用你的模型 能夠正確判別出多少瑕疵

	之後 再用同一個模型 更改路徑為 OK Perfect & OK Perfect CAM 看看 你能準確判斷多少

	OK Perfect  = 所有 OK 的電路板照片

	OK Perfect CAM  = 所有 OK 的電路板設計稿
